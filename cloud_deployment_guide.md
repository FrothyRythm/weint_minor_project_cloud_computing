# AWS Cloud Infrastructure & Deployment Guide

This guide walks you through setting up AWS S3, configuring CORS/IAM permissions, provisioning an EC2 instance, installing required software, configuring Nginx as a reverse proxy, setting up SSL, and managing the process daemon with PM2.

---

## Part 1: Amazon S3 Configuration

### 1. Create S3 Bucket
1. Go to the AWS S3 Console and click **Create bucket**.
2. Give it a unique name (e.g., `cloud-vault-storage-xyz`) and select your region.
3. Keep **Block all public access** enabled (checked). This keeps your storage files private. Files are downloaded via secure, temporary presigned URLs.

### 2. Configure CORS Policy
CORS allows client web browsers to perform direct `PUT` uploads to S3.
1. Click on the bucket name, go to the **Permissions** tab.
2. Scroll to the **Cross-origin resource sharing (CORS)** section and click **Edit**.
3. Paste the following JSON configuration (change origins to match your domain name):
```json
[
  {
    "AllowedHeaders": ["*"],
    "AllowedMethods": ["PUT", "GET"],
    "AllowedOrigins": [
      "http://localhost:5500",
      "http://localhost:3000",
      "https://your-domain.com"
    ],
    "ExposeHeaders": []
  }
]
```

### 3. Create IAM User and Policy
1. Go to the IAM console and click **Users** -> **Create user**.
2. Name the user (e.g., `cloud-vault-app`). Select **Attach policies directly** -> **Create policy**.
3. In the policy JSON editor, paste the following policy restricting accesses to your bucket:
```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:PutObject",
        "s3:GetObject",
        "s3:DeleteObject"
      ],
      "Resource": "arn:aws:s3:::your-s3-bucket-name/*"
    }
  ]
}
```
4. Complete creation, attach it to your user, and generate an **Access Key / Secret Key** in the user's security credentials tab. Save these credentials securely.

## Part 2: AWS EC2 Ubuntu Server Setup

### 1. Provision EC2
1. Launch a `t2.micro` or `t3.micro` instance running Ubuntu 22.04 LTS (under Free Tier).
2. In the Security Group configuration, ensure you add rules for:
   - Port 22 (SSH)
   - Port 80 (HTTP)
   - Port 443 (HTTPS)

### 2. Connect and Install Dependencies
Connect to your server via SSH:
```bash
ssh -i /path/to/key.pem ubuntu@your-ec2-public-ip
```

Update system software and install Node.js, Git, Nginx, and SQLite3:
```bash
sudo apt update && sudo apt upgrade -y
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs git nginx sqlite3
```

## Part 3: Deploy Application Code

### 1. Clone Code base
Clone your repository onto the server:
```bash
git clone https://github.com/your-username/your-repo.git /var/www/cloud-vault
cd /var/www/cloud-vault/backend
npm install
```

### 2. Configure Environment Variables
Create the backend configuration file:
```bash
cp .env.example .env
nano .env
```
Fill in the values:
- `PORT=3000`
- `FRONTEND_URL=https://your-domain.com` (or your EC2 public IP/DNS if no custom domain)
- `JWT_SECRET=your_super_secure_random_key`
- `AWS_REGION=us-east-1`
- `AWS_ACCESS_KEY_ID=your_iam_access_key`
- `AWS_SECRET_ACCESS_KEY=your_iam_secret_key`
- `S3_BUCKET_NAME=your-s3-bucket-name`

## Part 4: Process Management with PM2

PM2 ensures the server runs continuously in the background and restarts automatically on crash or server reboot.
```bash
sudo npm install -g pm2
pm2 start server.js --name "cloud-vault-api"
pm2 save
pm2 startup systemd
```
Copy and run the configuration script command generated by the `pm2 startup` terminal output.

## Part 5: Nginx Reverse Proxy & SSL Setup

### 1. Configure Nginx
Create an Nginx configuration file for your domain:
```bash
sudo nano /etc/nginx/sites-available/cloud-vault
```
Paste this configuration:
```nginx
server {
    listen 80;
    server_name your-domain.com www.your-domain.com;

    location / {
        root /var/www/cloud-vault/frontend;
        index index.html;
        try_files $uri $uri/ =404;
    }

    location /api {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```
Enable the configuration and restart Nginx:
```bash
sudo ln -s /etc/nginx/sites-available/cloud-vault /etc/nginx/sites-enabled/
sudo rm /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl restart nginx
```

### 2. Install Let's Encrypt Certbot (SSL)
Secure your web application with free HTTPS:
```bash
sudo apt install certbot python3-certbot-nginx -y
sudo certbot --nginx -d your-domain.com -d www.your-domain.com
```
Follow the interactive prompt to obtain the SSL certificate. Certbot will automatically rewrite the Nginx files to redirect HTTP to HTTPS.
